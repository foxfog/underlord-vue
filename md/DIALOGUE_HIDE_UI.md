# Система автоматического скрытия UI при диалоге

## Описание

Система **автоматически** скрывает определённые UI-элементы (кнопки, панели) при активном диалоге, независимо от того какая история загружена. Конфигурация устанавливается один раз в коде и действует на все диалоги.

## Как использовать

### Шаг 1: Отредактируйте конфигурацию

**Файл:** [src/renderer/constants/dialogue.js](src/renderer/constants/dialogue.js)

```javascript
export const DIALOGUE_HIDE_UI_CONFIG = [
	'hotbar',
	'map-button'
]
```

Просто добавьте/удалите элементы в этот массив. Готово! ✓

### Шаг 2: Готово!

Теперь при любом диалоге элементы из `DIALOGUE_HIDE_UI_CONFIG` будут автоматически скрываться.

## Примеры конфигурации

### Пример 1: Скрыть несколько элементов

```javascript
export const DIALOGUE_HIDE_UI_CONFIG = [
	'hotbar',           // нижняя панель
	'map-button',       // кнопка карты
	'stats-button'      // кнопка статистики
]
```

Результат: Когда активен диалог — скрываются все три элемента. Когда диалога нет — они показываются.

### Пример 2: Скрыть все кроме диалога

```javascript
export const DIALOGUE_HIDE_UI_CONFIG = ['all']
```

Результат: При диалоге скрываются все UI элементы, остаётся только диалоговое окно.

### Пример 3: Отключить скрытие

```javascript
export const DIALOGUE_HIDE_UI_CONFIG = []
```

Результат: Никакие элементы не будут автоматически скрыты при диалоге.

## Доступные элементы для скрытия

- `hotbar` — нижняя панель инструментов
- `topbar` — верхняя панель (группирует все кнопки)
- `stats-button` — кнопка статистики
- `inventory-button` — кнопка инвентаря  
- `map-button` — кнопка карты
- `dialogue` — диалоговое окно
- `all` — все элементы UI

## Как это работает

1. **Инициализация**: При загрузке приложения, система запоминает конфигурацию из `DIALOGUE_HIDE_UI_CONFIG`.

2. **Применение скрытия**: Когда появляется диалог, нарация, титры или выборы — система автоматически скрывает указанные элементы.

3. **Восстановление**: Когда диалог исчезает и переходим к следующему шагу — система показывает эти элементы обратно (если они не были явно скрыты другими командами).

4. **Приоритет**: 
   - Если элемент явно скрыт через `"type": "ui", "action": "hide"` — остаётся скрыт
   - Если элемент не скрыт явно — система скрывает/показывает его в зависимости от наличия диалога

## Состояние системы

Вы можете проверить состояние системы в DevTools:

```javascript
// В консоли браузера
visualNovel.value.uiVisibility

// Результат:
{
  all: false,
  'stats-button': false,
  'inventory-button': false,
  'map-button': false,
  topbar: false,
  hotbar: false,
  dialogue: false,
  dialogueHideUI: ['hotbar', 'map-button'],  // Глобальная конфигурация
  hasDialogue: true  // true если сейчас активен диалог
}
```

## Советы и трюки

### Совет 1: Динамическое изменение во время игры

Если нужно изменить конфигурацию прямо во время игры:

```javascript
// Из компонента или кода
visualNovel.value.setDialogueHideUI(['hotbar', 'stats-button'])
```

### Совет 2: Отладка в разработке

```javascript
// В консоли браузера смотрите:
visualNovel.value.uiVisibility.hasDialogue     // true если диалог активен
visualNovel.value.uiVisibility.dialogueHideUI  // текущая конфигурация
```

### Совет 3: Комбинация с явным скрытием

```json
{
  "type": "ui",
  "action": "hide",
  "target": ["inventory-button"]
},

{
  "type": "dialogue",
  "character": "mc",
  "text": "Сейчас скрыты: hotbar (система диалог-скрытия) + inventory-button (явное скрытие)"
}
```

Результат: При диалоге скрываются hotbar (системой) + inventory-button (явно).

## Отладка

Если элементы не скрываются:

1. Проверьте консоль браузера (F12) для логов

2. Убедитесь что элементы в `DIALOGUE_HIDE_UI_CONFIG` имеют правильные ID

3. Проверьте `visualNovel.value.uiVisibility.hasDialogue` — должно быть `true` при активном диалоге

4. Если элемент явно скрыт через `"type": "ui"` — он останется скрыт даже после диалога

## Часто задаваемые вопросы

**Q: Почему нужно редактировать константу в коде?**
A: Потому что это глобальная настройка приложения, которая применяется на все диалоги независимо от истории. Не нужно вписывать это в каждую историю.

**Q: Почему элемент остался скрыт после диалога?**
A: Вероятно, он был явно скрыт через `"type": "ui", "action": "hide"` команду. Система скрытия диалога не переопределяет явное скрытие.

**Q: Можно ли менять конфигурацию во время игры?**
A: Да, вызовите `visualNovel.value.setDialogueHideUI(['новые', 'элементы'])` из кода JavaScript. Но после перезагрузки истории вернётся к дефолтной конфигурации из `DIALOGUE_HIDE_UI_CONFIG`.

**Q: Сохраняется ли конфигурация в save?**
A: Нет, конфигурация не сохраняется. При загрузке сохранения используется дефолтная конфигурация из `DIALOGUE_HIDE_UI_CONFIG`.

**Q: Что если я хочу разную конфигурацию для разных историй?**
A: Используйте `setDialogueHideUI()` в коде, когда нужно переключиться на другую историю. Или создайте несколько вспомогательных функций для разных "режимов".
