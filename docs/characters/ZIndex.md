# Z-Index для частей тела персонажа

Свойство `zindex` (или `z-index`) определяет порядок наложения частей тела персонажа. Это необходимо когда части тела перекрывают друг друга и нужно контролировать какая часть находится спереди.

## Синтаксис

В JSON файле части тела (`body.json`) используйте `zindex` (без дефиса):

```json
{
  "arm_right": {
    "image": "images/sprites/characters/mc/arm_right.png",
    "parent": "body",
    "zindex": -1,
    "offset": {
      "x": -19.2,
      "y": -43
    }
  }
}
```

> [!TIP]
> Рекомендуется использовать `zindex` (без дефиса), так как это каноничный синтаксис JSON. Однако `z-index` также поддерживается компонентом `SpritePart.vue`.

## Значения

- **Положительные значения** (1, 2, 10) - части находятся поверх остальных.
- **Ноль** (0, или отсутствие свойства) - обычный порядок (значение по умолчанию). Отрисовка идет после фоновых частей, например `body`.
- **Отрицательные значения** (-1, -2) - части находятся под остальными.

## Примеры использования

### 1. Правая рука позади туловища
```json
{
  "arm_right": {
    "zindex": -1
  }
}
```
Рука будет отрисована позади тела.

### 2. Волосы поверх головы
```json
{
  "hair": {
    "parent": "head",
    "zindex": 1
  }
}
```
Волосы будут отрисованы поверх головы.

### 3. Левая рука впереди (По умолчанию)
```json
{
  "arm_left": {
    "zindex": 0
  }
}
// или просто не указывать zindex:
{
  "arm_left": {
    "image": "..."
  }
}
```

## Порядок отрисовки в иерархии

```json
{
  "body": {
    "image": "..."
  },
  "arm_left": {
    "parent": "body"
  },
  "arm_right": {
    "parent": "body",
    "zindex": -1
  }
}
```

**Результат:**
1. `arm_right` - z-index -1 (позади всех, отрисуется первым)
2. `body` - z-index 0 (фон)
3. `arm_left` - z-index 0 (по умолчанию, отрисуется после body)

Порядок: `arm_right` → `body` → `arm_left`

> [!NOTE]
> Помните про иерархию: родительская часть влияет на отрисовку дочерних. Если у `body` стоит z-index отличный от нуля, он повлияет и на все прикрепленные части.
