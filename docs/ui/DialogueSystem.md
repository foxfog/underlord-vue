# Система Диалогов

Система диалогов включает в себя две важные особенности: многошаговые диалоги и автоматическое скрытие элементов UI.

## Многошаговые диалоги (Multi-Step Dialogue)

Многошаговые диалоги позволяют разделить текст одного диалогового блока на несколько шагов, которые отображаются по очереди при нажатии кнопки "Продолжить". Текст каждого шага накапливается и отображается вместе с предыдущими шагами.

### Синтаксис

```json
{
  "type": "dialogue",
  "id": "unique_id",
  "steps": [
    { "text": "Первый текст" },
    { "text": "Второй текст" },
    { "text": "Третий текст" }
  ]
}
```

### Примеры

**С персонажами:**
```json
{
  "type": "dialogue",
  "id": "dialogue_sequence",
  "steps": [
    { "character": "mc", "text": "Кто-то...?" },
    { "character": "npc", "text": "Это я! Помните?" },
    { "text": "Их голоса эхом отозвались в пустом здании" }
  ]
}
```

**С действиями между шагами:**
```json
{
  "type": "dialogue",
  "steps": [
    { "text": "Первый текст" },
    { "type": "action", "action": "play_sound_effect" },
    { "text": "Текст после действия" }
  ]
}
```

### Поведение и Typewriter эффект
- **Накопление текста**: Каждый новый шаг добавляется к предыдущим с разделителем `<br>`.
- При включенном медленном выводе текста (Typewriter эффект) печать продолжается только для нового текста, не переписывая уже напечатанное.
- Шаги без явного типа обрабатываются как текстовые (`"type": "text"`).

## Автоматическое скрытие UI

Система автоматически скрывает определённые UI-элементы (кнопки, панели) при активном диалоге (narration, choices, title, dialogue).

### Конфигурация

Конфигурация устанавливается один раз и действует на все диалоги:
**Файл:** `src/renderer/constants/dialogue.js`

```javascript
export const DIALOGUE_HIDE_UI_CONFIG = [
  'hotbar',
  'map-button'
]
```

При любом диалоге элементы из массива будут автоматически скрыты, а после диалога — показаны.

### Доступные элементы для скрытия
- `hotbar` — нижняя панель инструментов
- `topbar` — верхняя панель (группирует все кнопки)
- `stats-button` — кнопка статистики
- `inventory-button` — кнопка инвентаря  
- `map-button` — кнопка карты
- `dialogue` — диалоговое окно
- `all` — все элементы UI

### Динамическое изменение

Если нужно изменить конфигурацию прямо во время игры из компонента:
```javascript
visualNovel.value.setDialogueHideUI(['hotbar', 'stats-button'])
```
> [!NOTE]
> После перезагрузки истории или сейва конфигурация сбрасывается до дефолтной из `DIALOGUE_HIDE_UI_CONFIG`. Состояние системы не сохраняется в save-файлах.

### Приоритет скрытия
Если элемент явно скрыт через шаг истории:
```json
{ "type": "ui", "action": "hide", "target": ["inventory-button"] }
```
он останется скрытым и после диалога. Система диалогового скрытия не переопределяет явное скрытие.
