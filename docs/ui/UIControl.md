# Система управления видимостью UI (UI Control System)

Эта система позволяет контролировать видимость UI-элементов (кнопок, панелей) напрямую из JSON-файлов истории. Это особенно полезно для кинематографичных сцен или заставок (intro).

## Синтаксис

В JSON истории используется шаг типа `"ui"`.

```json
{
  "type": "ui",
  "action": "show" | "hide",
  "target": ["element-id-1", "element-id-2"]
}
```

### Параметры

- **type** (обязательно): `"ui"`
- **action** (обязательно): `"show"` или `"hide"`
- **target** (опционально): массив ID элементов.
  - Оставлен пустым (или отсутствует) — применяется ко ВСЕМ элементам.
  - `"all"` — специальный ключ для всех элементов.

## Доступные ID элементов

- `"stats-button"` — кнопка статистики (📊 Статы)
- `"hotbar"` — нижняя панель
- `"dialogue"` — блок диалогового окна
- `"all"` — все UI элементы

## Примеры использования

**Скрыть всё:**
```json
{
  "type": "ui",
  "action": "hide"
}
```

**Скрыть всё, а потом показать только диалог:**
```json
{
  "type": "ui",
  "action": "hide",
  "target": ["all", "dialogue"]
}
```
*(Система сначала применит действие "all", а затем исключит из него "dialogue", создав логику "Скрыть всё, кроме диалога")*

**Вернуть все UI элементы:**
```json
{
  "type": "ui",
  "action": "show"
}
```

## Как это работает под капотом

- В `useVisualNovel.js` используется объект `uiVisibility`, хранящий состояние (`true`/`false`) каждого элемента.
- Шаг `"ui"` вызывает функцию `handleUIStep(step)`.
- Состояние передаётся в `Game.vue`, где элементы рендерятся через `v-if`. Они полностью удаляются из DOM, когда скрыты.

> [!NOTE]
> Скрытие элементов сохраняется до тех пор, пока вы не вызовете команду `"show"`. Скрытие через этот механизм не сбрасывается диалогами (в отличие от `DIALOGUE_HIDE_UI_CONFIG`). Состояние скрытия UI **не сохраняется** в файлы сохранений игры, так как это визуальное состояние сцены.
