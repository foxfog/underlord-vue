# Game Styles Structure

Этот документ описывает организацию стилей для игровых компонентов.

## Структура файлов

### `game.css`
Главный файл, который импортирует все стили игровых компонентов с красивыми вложенностями (nesting).

**Импортируемые файлы:**
- `visual-novel.css` - Стили для компонентов визуального романа
- `character.css` - Стили для отображения персонажей
- `character-stats-modal.css` - Стили для модального окна статистики персонажа
- `text-input-modal.css` - Стили для модального окна ввода текста
- `equip-part.css` - Стили для отображения экипировки

### `visual-novel.css`
Содержит стили для компонентов:
- `Background.vue` - фоновое изображение
- `CharacterList.vue` - список персонажей
- `DialogueBox.vue` - диалоговое окно с репликами и выборами
- `TitleBlock.vue` - блок заголовка

**Основные селекторы:**
- `.background-img` - фоновое изображение
- `.title-box` - контейнер заголовка
- `.title-content` - содержимое заголовка
- `.characters` - контейнер для персонажей
- `.dialogue-box` - диалоговое окно
  - `.speaker` - имя говорящего
  - `.dialogue-text` - текст диалога
  - `.narration-text` - нарративный текст
  - `.choices` - контейнер выборов
    - `.choice-btn` - кнопка выбора
  - `.continue-btn` - кнопка продолжения

### `character.css`
Содержит стили для компонента:
- `Character.vue` - отображение персонажа с анимацией и частями тела

**Основные селекторы:**
- `.char` - контейнер персонажа
  - `.char-body` - тело персонажа
    - `.char-part` - отдельная часть (голова, туловище, конечности)
      - `.char-part-sprite` - спрайт части
        - `.sprite-image` - изображение спрайта

### `character-stats-modal.css`
Содержит стили для компонента:
- `CharacterStatsModal.vue` - модальное окно со статистикой персонажа

**Основные селекторы:**
- `.modal` - контейнер модала
  - `.modal-content` - содержимое модала
    - `.modal-header` - заголовок с кнопкой закрытия
    - `.modal-body` - основное содержимое
      - `.stats-grid` - сетка статистики
        - `.stat-item` - элемент статистики
          - `.stat-label` - подпись статистики
          - `.stat-value` - значение статистики
          - `.stat-bar` - полоса прогресса
            - `.stat-fill` - заполненная часть полосы
            - `.stat-fill.mp` - специальный стиль для магии
    - `.modal-footer` - подвал с кнопками

### `text-input-modal.css`
Содержит стили для компонента:
- `TextInputModal.vue` - модальное окно для ввода текста

**Основные селекторы:**
- `.modal` - контейнер модала
  - `.modal-content` - содержимое модала
    - `.modal-header` - заголовок с кнопкой закрытия
    - `.modal-header-no-close` - альтернативный заголовок без кнопки
    - `.modal-body` - основное содержимое
      - `.input-description` - описание для пользователя
      - `.text-input` - поле ввода текста
    - `.modal-footer` - подвал с кнопками

### `equip-part.css`
Содержит стили для компонента:
- `EquipPart.vue` - отображение экипировки персонажа

**Основные селекторы:**
- `.equip-part` - контейнер экипировки
  - `.equip-part-sprite` - спрайт экипировки
    - `.sprite-image` - изображение спрайта

## Переменные и их использование

Стили используют переменные из `UI/_root.css`:
- `--color-primary` - основной цвет (#4CAF50)
- `--color-primary-light` - светлый основной цвет
- `--color-secondary` - дополнительный цвет (#FF9800)
- `--color-white` - белый цвет
- `--color-black` - чёрный цвет
- `--color-shadow` - цвет теней
- `--color-border` - цвет границ

Для персонажей используются специальные переменные:
- `--char-height` - высота персонажа (вычисляется динамически)
- `--charspriteH` - высота спрайта части
- `--charbodyspriteH` - высота спрайта тела
- `--appW` - ширина приложения
- `--appH` - высота приложения

## Особенности реализации

### CSS Nesting
Все файлы используют современный синтаксис CSS Nesting для лучшей организации и читаемости:
```css
.parent {
	/* стили parent */
	
	.child {
		/* стили child */
	}
}
```

### Переходы и эффекты
- Диалоговые компоненты используют плавные переходы (`transition`)
- Кнопки имеют hover эффекты для лучшей пользовательской опыт
- Модальные окна имеют полупрозрачный фон для выделения

### Адаптивность
- Стили используют относительные единицы (em, %) для лучшей адаптации к размерам экрана
- Модальные окна центрируются на экране и имеют max-width для мобильных устройств

## Интеграция со стилями проекта

Все стили импортируются в главный файл `src/renderer/public/styles/main.css`:
```css
@import './game/game.css';
```

Это обеспечивает согласованность с остальными стилями приложения и переменными из `UI/_root.css`.
